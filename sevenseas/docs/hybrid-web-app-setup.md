# Web + App 동시 대응 구조

## 현재 적용
- Vite + Phaser 단일 코드베이스
- PWA 매니페스트/서비스워커 구성 (`vite-plugin-pwa`)
- 터치 입력 UI 추가 (모바일/앱 래퍼 대응)
- 키보드 + 터치 입력을 하나의 `controlState`로 통합

## 빌드 타깃
- 웹 배포: `npm run build` 결과물(`dist`) 배포
- 설치형 웹앱(PWA): 모바일 브라우저에서 "홈 화면에 추가"
- 앱 래퍼: 동일 `dist`를 Tauri/Capacitor WebView에 로드

## 앱 래퍼 전환 가이드
1. PWA 단계에서 게임 로직/입력/저장소를 웹 표준 API 기반으로 유지
2. 네이티브 기능이 필요해지면 Tauri(데스크톱) 또는 Capacitor(모바일) 래퍼만 추가
3. 네이티브 API는 `platform/` 어댑터 레이어를 만들어 게임 도메인과 분리

## 주의
- 원본 상용 게임 아트/사운드는 저작권 이슈가 있으므로 그대로 사용 금지
- 현재 그래픽은 코드 생성 텍스처 기반의 유사 분위기 프로토타입
